# Change history for ui-erm-usage

## 2.0.0 (IN PROGRESS)
* Upload XMl encoded Counter 4 reports
* Ability to download counter reports as CSV for single and multiple months (for Counter 4 JR1 only)
* Ability to start harvester for a single usagedata provider
* Hide credentials in front end (e.g. for presentations)
* Display info when harvester failed to download certain reports
* Ability to delete counter reports
* Improve search, sort and filter capabilities
* Filterconfig of SearchAndSort component is dynamic dependent on aggregator definitions
